Automatic Workspace Creation

When a new user signs up or logs in for the first time, the system checks for an existing workspace. If none exists, a Personal Workspace is automatically provisioned with the user as Owner. This is handled in auth-actions.ts server actions.

// auth-actions.ts — runs after every successful login
const { data: orgs } = await supabase
  .from('org_members')
  .select('org_id')
  .eq('user_id', data.user.id)
  .limit(1);

if (!orgs || orgs.length === 0) {
  // Auto-provision Personal Workspace
  const { data: newOrg } = await supabase
    .from('organizations')
    .insert({ name: 'Personal Workspace', slug })
    .select().single();

  await supabase.from('org_members')
    .insert({ org_id: newOrg.id, user_id, role: 'owner' });
}
✓
All project data is isolated at the database level using Supabase Row Level Security (RLS). Members can only query projects that belong to organizations they are members of.

Agent Long-Term Memory

The agent remembers your preferences, tech stack, and past generations — improving every build.

🧠

Semantic Recall

Past prompts are embedded as 1536-dim vectors. Before each generation, the agent performs a cosine similarity search to find the most relevant past context.

📚

Preference Learning

Tech stack choices, folder structures, and integration patterns from past high-rated generations are automatically surfaced as context hints.

⚡

Context Injection

Recalled memories are injected directly into the planSpec system prompt before the LLM plans the application structure.

🔐

User-Scoped

Memories are stored per-user and per-organization. Your team's context never bleeds into another team's generations.

Database: generation_memory

-- pgvector-backed memory table
CREATE TABLE generation_memory (
  id          UUID           PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID           REFERENCES auth.users(id),
  prompt      TEXT           NOT NULL,
  embedding   vector(1536),   -- OpenAI text-embedding-3-small
  tech_stack  TEXT[],
  metadata    JSONB,
  created_at  TIMESTAMPTZ    DEFAULT NOW()
);

CREATE INDEX ON generation_memory
  USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 100);
⚠
Long-term memory requires OPENAI_API_KEY to generate embeddings. Without it, the system falls back to simple keyword-based context recall from the most recent 3 projects.

Real-Time Collaboration

Multiple users can co-generate in the same project session with live presence and delta streaming.

👥

Presence Tracking

See who else is active in a project workspace in real time using Supabase Presence channels.

⚡

Generation Broadcast

Every streamed generation delta is broadcast to all channel subscribers — team members see code appearing live.

Implementation: src/lib/collaboration.ts

const channel = supabase.channel(`project:${projectId}`, {
  config: {
    broadcast: { self: false },
    presence: { key: projectId }
  }
});

// Subscribe to live deltas from teammates
channel.on('broadcast', { event: 'delta' }, ({ payload }) => {
  appendToSharedBuffer(payload.delta);
}).subscribe();

// Broadcast your own generation deltas
channel.send({ type: 'broadcast', event: 'delta', payload: { delta } });

Semantic Code Search

Search inside all generated projects using natural language — powered by pgvector.

Automatic Indexing

Every file generated is automatically indexed by codeSearch.indexProject() immediately after a project is persisted to the database. No manual indexing step is required.

Search SQL Function

-- Searches code_chunks using cosine similarity
SELECT * FROM search_code_chunks(
  query_embedding := '[0.12, 0.45, ...]'::vector,
  match_threshold := 0.75,
  match_count     := 10,
  p_org_id        := 'your-org-uuid'
);

The root layout.tsx reads the incoming host header on every request and resolves a brand config via whiteLabelManager.resolveConfig(host). If a config is found, CSS custom properties are injected as an inline <style> block:

Migration Guide

Steps to upgrade an existing Phase 7 deployment to Phase 8.

  1. Run the SQL migration — copy the Phase 8 block from supabase/schema.sql and run it in Supabase SQL Editor. This adds all new tables and enables pgvector.
  2. Add environment variables — set E2B_API_KEY, OPENAI_API_KEY, and CRON_SECRET in Vercel.
  3. Redeploy — push to main or trigger a manual Vercel deployment. The vercel.json cron configuration activates automatically.
  4. Verify cron jobs — check Vercel Dashboard → Settings → Cron Jobs to confirm the three scheduled jobs appear.
  5. Test workspace provisioning — log out and log back in. Confirm a "Personal Workspace" entry appears in your organizations table in Supabase.
  6. Optional: enable pgvector index — for large deployments, run ANALYZE generation_memory; after inserting the first 1000 rows to optimize the IVFFlat index.
✓
Phase 8 is fully backward compatible. All Phase 1–7 features and API routes continue to work unchanged. The only breaking change is the renaming of the mode toggle in the UI (Single Unit / Web Suite / Mobile App replaces the previous Single / Multi-file toggle).
