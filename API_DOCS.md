# API Documentation

## Overview

The AI App Builder provides a comprehensive REST API for all 18 phases of autonomous application development.

## Base URL

```
https://your-app.vercel.app/api
```

## Authentication

Most endpoints use Supabase authentication. Include the session token in the `Authorization` header:

```http
Authorization: Bearer YOUR_SUPABASE_TOKEN
```

## Core Endpoints

## Phase 1-2: Code Generation

### POST /api/generate
Generate code from natural language prompts or images.

**Request Body:**
```json
{
  "prompt": "Create a login form with email and password fields",
  "imageUrl": "https://example.com/screenshot.png",
  "stream": false,
  "multiFile": true,
  "mode": "web-app"
}
```

**Parameters:**
- `prompt` (string, optional): Natural language description
- `imageUrl` (string, optional): URL of screenshot for vision-to-code
- `stream` (boolean, default: false): Enable streaming response
- `multiFile` (boolean, default: false): Generate multi-file app
- `mode` (enum): `"web-component"` | `"web-app"` | `"mobile-app"` | `"vision"`

**Response:**
```json
{
  "files": {
    "app/page.tsx": "export default function Page() { ... }",
    "components/LoginForm.tsx": "..."
  },
  "description": "A login form component",
  "schema": null,
  "integrations": [],
  "id": "uuid",
  "timestamp": 1704600000000
}
```

**Streaming Response:**
```http
Content-Type: text/event-stream

data: {"type":"chunk","delta":"export"}
data: {"type":"chunk","delta":" default"}
data: {"type":"result","result":{...}}
```

## Phase 4: Deployment & Export

### POST /api/deploy
Deploy to Vercel.

**Request Body:**
```json
{
  "projectId": "uuid",
  "files": { "app/page.tsx": "..." },
  "name": "my-app",

```

**Response:**

**Request Body:**

## Error Responses

All endpoints may return error responses:

```json
{
  "error": "Error message",
  "code": "ERROR_CODE",
  "details": { ... }
}
```

Common HTTP status codes:
- `400` - Bad Request (validation error)
- `401` - Unauthorized (authentication required)
- `403` - Forbidden (insufficient permissions)
- `404` - Not Found
- `429` - Too Many Requests (rate limited)
- `500` - Internal Server Error
- `503` - Service Unavailable (dependency down)

## Rate Limiting

- Unauthenticated: 10 requests/minute
- Authenticated: 100 requests/minute
- Enterprise: Unlimited

## SDK Examples

### JavaScript/TypeScript

```typescript
// Generate code
const response = await fetch('/api/generate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    prompt: 'Create a button component',
    multiFile: false
  })
});
const result = await response.json();

// Deploy to Vercel
const deployResponse = await fetch('/api/deploy', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    projectId: result.id,
    files: result.files
  })
});
const deployment = await deployResponse.json();
```

### Python

```python
import requests

# Generate code
response = requests.post('/api/generate', json={
    'prompt': 'Create a login form',
    'multiFile': True
})
result = response.json()

# Validate idea
validation = requests.post('/api/validate-idea', json={
    'idea': 'AI-powered analytics'
})
print(validation.json())
```
